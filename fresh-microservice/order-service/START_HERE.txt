## 🛠️ THE FIX EXPLAINED

### What Was Wrong
```
Entity says:    "This field can be NULL"
Database says:  "This field MUST have a value"
Result:         When inserting without a value → ERROR!
```

### What's Fixed Now
```
Entity says:    "This field can be NULL"
Database says:  "This field can be NULL (DEFAULT NULL)"
Result:         When inserting without a value → Works! ✅
```

### The One-Line Fix
```java
// Line 46 in OrderItem.java
@Column(name = "price_at_purchase", nullable = true, precision = 10, scale = 2,
        columnDefinition = "DECIMAL(10, 2) DEFAULT NULL")
private BigDecimal priceAtPurchase;
```

## 💡 WHY THIS HAPPENS (And How to Prevent)

### The Problem Pattern
This happens when:
1. JPA entity says "nullable = true" (field can be NULL)
2. Database column is created as "NOT NULL" (no default)
3. Code tries to insert NULL → Database rejects it

### The Prevention
Always use:
```java
@Column(nullable = true, columnDefinition = "TYPE DEFAULT NULL")
// OR
@Column(nullable = false, columnDefinition = "TYPE DEFAULT value")
// OR
Initialize in @PrePersist
```

Read: **DATABASE_ISSUE_FIX.md** section "Best Practices"
